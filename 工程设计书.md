# SVN AI Review V2.0 — 工程设计书

## 1. 项目概述

### 1.1 背景

SVN AI Review 是一款面向 Windows 内网环境的 SVN 代码自动审查工具。通过对 SVN 仓库提交记录进行自动扫描和差异提取，结合**规则引擎 + AI 大模型 + 可定制 Skill** 三层审查体系分析代码质量，生成结构化日报并通过 Teams/Email 推送，帮助 UEFI/EDK2 等固件开发团队实现代码审查自动化。

### 1.2 版本变化

| 维度 | V1.4 | V2.0 |
|------|------|------|
| 架构 | 单文件 | 模块化分层（37 py 文件 + Next.js 前端） |
| 审查引擎 | 规则 + mock LLM | 规则 + AI + Skill 三层可插拔 |
| 前端 | 原生 HTML | Next.js 14 + Tailwind CSS + 8 页面 SPA |
| 扫描性能 | 串行阻塞 | ThreadPoolExecutor 并发 + 快速预检跳过 |
| 容错 | 无 | SVN 重试 + LLM 重试 + IntegrityError 防御 + WAL 写锁 |
| 日志 | 无 | TimedRotatingFileHandler，30 天保留 |
| 通知 | Teams | Teams + Email 双渠道 |
| 配置 | 手动编辑 YAML | Web UI 全量配置 + config.example 模板 |
| 安全 | 无认证 | 可选 API Key 中间件 + 路径遍历防御 + URL 校验 |
| 部署 | 单 bat | dev / 构建 / StartTool 三脚本 + config.example 模板 |

### 1.3 约束条件

- **部署环境**：Windows 单机，仅需 Python 3.12+ + SVN CLI，无 Node.js 运行时
- **数据库**：SQLite（WAL 模式），单机部署足够
- **SVN 认证**：本地 `file://` 跳过证书校验，远程需用户名密码
- **LLM 可选**：支持云端 API（DeepSeek/OpenAI）和本地模型（Ollama），本地零费用
- **内网兼容**：无外网时可纯本地运行（Ollama + file:// SVN）

## 3. 架构设计

### 3.1 分层架构

```
┌────────────────────────────────────────────┐
│  前端 (Next.js 静态导出)                     │
│  app/static/ ← npm run build 产物          │
│  开发模式下 Next.js HMR :3000               │
├────────────────────────────────────────────┤
│  API 中间件层（可选 API Key 认证 + CORS）    │
├────────────────────────────────────────────┤
│  路由层 (app/routes/ 8 模块)                │
│  dashboard / projects / scan / schedule /  │
│  reports / issues / authors / settings     │
├────────────────────────────────────────────┤
│  服务层 (app/services/)                     │
│  svn_client / scanner / scan_progress /     │
│  review(rule/llm/skill) / report_builder /  │
│  notifier                                   │
├────────────────────────────────────────────┤
│  数据层 (app/models/ 5 表 + database)       │
│  SQLAlchemy ORM + SQLite WAL + 索引        │
├────────────────────────────────────────────┤
│  基础设施                                    │
│  config(mtime缓存) / logging / scheduler   │
└────────────────────────────────────────────┘
```

### 3.2 数据流（并发扫描）

```
[UI手动 / 定时触发] → scan_start
  → collect_commits()           # SVN log 查询（秒级）
  → ThreadPoolExecutor(3)       # 并发处理
    ├── svn_diff_summarize()    # 快速预检文件数
    ├── svn_diff_safe()         # 获取完整 diff
    ├── rule_engine.review()    # 规则审查（毫秒）
    ├── llm_review()            # AI 通用审查（秒级）
    └── skill_engine.review()   # Skill 审查（提示词驱动）
  → _write_lock                 # 仅串行 DB 写入（毫秒）
  → create_report()             # Markdown 报告生成
  → send_notification()         # Teams + Email 推送
```

### 3.3 审查引擎三层架构

| 层级 | 引擎 | 速度 | 检测能力 | 可配置性 |
|------|------|------|---------|---------|
| L1 | rule_engine | 毫秒 | 合并冲突/TODO/调试输出/memcpy | Web UI 开关 |
| L2 | llm.py | 秒级 | 语义理解、逻辑错误 | 提供商可插拔 |
| L3 | skill_engine | 秒级 | 任何用户定义的审查维度 | YAML 配置 Skill 提示词 |

### 3.4 模块清单

| 模块 | 文件 | 职责 |
|------|------|------|
| 入口 | `app/main.py` | FastAPI 创建、API Key 中间件、CORS、catch-all SPA 路由、启动/关闭事件 |
| 配置 | `app/config.py` | env + YAML 加载、mtime 缓存、深度合并默认值、原子写、校验 schedule |
| 数据库 | `app/database.py` | SQLite WAL + busy_timeout + FK ON、SessionLocal、init_db |

| 调度 | `app/scheduler.py` | APScheduler 多 job 管理、reload 机制 |
| 模型 (5) | `app/models/` | SvnCommit、ChangedFile、ReviewIssue、DailyReport、AuthorStat |
| Schema (5) | `app/schemas/` | Project、Scan、Schedule、LLM、Settings（含 validator） |

## 4. 数据库设计

### 4.1 E-R 关系

```
svn_commits (1) ──── (N) changed_files    [FK, ondelete CASCADE]
       │
       │ (project_name + revision)          [联合索引]
       ↓
review_issues                              [level 索引, engine_type 索引]

daily_reports (1) ──── (N) author_stats   [report_id 关联]
```

### 4.2 索引设计

| 表 | 索引 | 用途 |
|----|------|------|
| svn_commits | UNIQUE(project_name, revision) | 去重扫描 |
| svn_commits | author | 作者统计查询 |
| svn_commits | commit_time | 按时间排序 |
| changed_files | commit_id | JOIN 查询 |
| review_issues | level | 按级别筛选 |
| review_issues | (project_name, revision) | 报告生成 |
| review_issues | engine_type | 引擎类型筛选 |

### 4.3 表结构

#### svn_commits

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | |
| project_name | VARCHAR NN | 项目名 |
| revision | VARCHAR NN | SVN 版本号 |
| author | VARCHAR INDEX | 提交人 |
| commit_time | DATETIME INDEX | 提交时间 |
| message | TEXT | 提交信息 |
| changed_file_count | INTEGER | 变更文件数 |

#### changed_files

| 字段 | 类型 | 说明 |

## 6. 配置设计

### 6.1 `config.yaml` 顶层结构

```yaml
timezone: Asia/Shanghai
schedule:          # 定时扫描（多时段）
scan:              # 扫描参数（文件上限/扩展名/排除路径/规则）
llm:               # LLM 配置（并发/重试/提供商列表）
skills:            # Skill 审查（提示词+文件类型+风险等级）
notifications:     # Teams + Email 通知
projects:          # SVN 项目白名单
```

### 6.2 Skill 配置格式

```yaml
skills:
  - id: uefi_pointer_check
    name: 指针安全检查
    level: P2
    enabled: true
    file_types: ['.c', '.h']
    prompt: |
      你是 UEFI BIOS 代码审查专家。检查指针安全...
      输出 JSON：{"issues":[{"line":N,"desc":"...","suggestion":"..."}]}
```

### 6.3 `.env` 密钥管理

所有敏感信息（API Key、SMTP 密码、SVN 密码）存储在 `.env` 中，`.gitignore` 保护。

```env
# SVN 认证
SVN_USERNAME=   SVN_PASSWORD=

# AI 提供商（api_key_ref 引用此变量）
DEEPSEEK_API_KEY=   OPENAI_API_KEY=

# 通知
TEAMS_WEBHOOK_URL=   EMAIL_SMTP_PASSWORD=

# 可选认证
API_KEY=           # 设置后所有 /api/ 请求需 X-API-Key 头
```

### 6.4 每台电脑独立配置

`config.yaml` 在 `.gitignore` 中。首次启动自动从 `config.example.yaml` 复制。clone 后每台电脑拥有独立配置。

## 7. 性能优化

| 优化项 | 状态 | 效果 |
|--------|------|------|
| config mtime 缓存 | ✅ | 消除数千次 YAML 解析 |
| `--summarize` 快速预检 | ✅ | 超大提交秒级跳过，不再超时 |
| ThreadPoolExecutor 并发 | ✅ | 3 线程并行，速度 2-3x |
| LLM Session 复用 | ✅ | 减少 TCP 握手 |
| LLM 指数退避重试 | ✅ | 网络波动不丢审查 |
| SQLite WAL + busy_timeout | ✅ | 并发写入不锁 |
| 写锁串行 DB 操作 | ✅ | LLM 审查并行、DB 写入安全 |
| 数据库索引 | ✅ | 查询加速 10-100x |
| need_review=0 截断到 200 字 | ✅ | DB 体积减 20-40% |

## 8. 安全加固

| 措施 | 生效条件 | 说明 |
|------|---------|------|
| 路径遍历防御 | 始终 | `resolve()` + 前缀校验 |
| API Key 认证 | `.env` 设 `API_KEY` 后 | 所有 `/api/` 需 `X-API-Key` 头 |
| CORS 中间件 | 始终 | 允许来源可配 `CORS_ORIGINS` |
| SVN URL scheme 校验 | 始终 | 拒绝非标准 scheme + `--` 注入 |
| LLM URL scheme 校验 | 始终 | 仅允许 `http://` / `https://` |
| api_key_ref 脱敏 | 始终 | Settings API 仅返回掩码值 |
| 日期格式校验 | 始终 | Pydantic validator 拒绝非法格式 |
