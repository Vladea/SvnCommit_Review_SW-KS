# SVN AI Review V2.0 — 工程设计书

## 1. 项目概述

### 1.1 背景

SVN AI Review 是一个面向 Windows 内网环境的 SVN 代码自动审查工具，通过定时扫描 SVN 仓库提交记录，提取变更差异，结合规则引擎与 AI 大模型进行自动化代码审查，生成每日审查报告并通过 Teams/邮件等渠道推送。

### 1.2 版本目标

| 维度 | V1.4（现状） | V2.0（目标） |
|------|------------|------------|
| 架构 | 单文件 297 行 | 模块化分层，26+ 文件 |
| 审查引擎 | 规则 + mock LLM | 规则 + 多 AI 接口可插拔 |
| 前端 | 原生 HTML/JS/CSS | Next.js + Tailwind + shadcn/ui |
| 日志 | 无 | TimedRotatingFileHandler，30天保留 |
| 容错 | 无重试 | SVN 操作可配置重试 |
| 通知 | Teams webhook | Teams + Email + 飞书预留 |
| 设置 | 需手动编辑 yaml | 全局 UI 设置页 + 项目级覆盖 |
| 定时任务 | 单一时段 | 多时段 + 独立通知配置 |

### 1.3 约束条件

- **部署环境**：Windows 内网单机，仅需 Python + SVN CLI，无 Node.js 运行时
- **数据库**：SQLite，单机部署足够
- **SVN 认证**：必填（内网 SVN 服务器需要用户名密码）
- **网络**：内网环境，可能无法访问公网 API（本地 Ollama 模型作为备选）

## 3. 架构设计

### 3.1 分层架构

```
┌──────────────────────────────────────┐
│  前端 (Next.js 静态导出)              │
│  app/static/ ← 构建产物              │
├──────────────────────────────────────┤
│  路由层 (app/routes/)                 │
│  dashboard / projects / scan /       │
│  schedule / reports / issues /       │
│  authors / settings                  │
├──────────────────────────────────────┤
│  服务层 (app/services/)               │
│  svn_client / scanner / review /     │
│  report_builder / notifier           │
├──────────────────────────────────────┤
│  数据层 (app/models/ + app/database)  │
│  SQLAlchemy ORM + SQLite            │
├──────────────────────────────────────┤
│  基础设施                              │
│  config / logging / scheduler        │
└──────────────────────────────────────┘
```

### 3.2 数据流

```
[定时触发 / UI手动] → scanner.run_range()
  → svn_client.svn_logs()        # SVN log 获取
  → svn_client.filter_by_date()  # 按真实日期过滤
  → svn_client.svn_diff_safe()   # 获取 diff
  → svn_client.split_diff()      # 拆分多文件 diff
  → svn_client.should_review()   # 文件过滤
  → rule_engine.review()         # 规则审查
  → llm_client.review()          # AI 审查
  → report_builder.create_report()  # 生成报告
  → notifier.send_*()            # 推送通知
```

## 4. 数据库设计

### 4.1 ER 图（5 张表）

```
svn_commits (1) ──→ (N) changed_files
       │
       │ (通过 project_name + revision 关联)
       ↓
review_issues

daily_reports (1) ──→ (N) author_stats
```

### 4.2 表结构

#### svn_commits — SVN 提交记录
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | |
| project_name | VARCHAR | 项目名 |
| revision | VARCHAR | SVN 版本号 |
| author | VARCHAR | 提交人 |
| commit_time | VARCHAR | 提交时间 |
| message | TEXT | 提交信息 |
| changed_file_count | INTEGER | 变更文件数 |
| scanned_at | DATETIME | 扫描时间 |
| **UNIQUE(project_name, revision)** | | 防止重复扫描 |

#### changed_files — 变更文件
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | |
| commit_id | INTEGER FK | 关联 svn_commits |
| file_path | TEXT | 文件路径 |
| diff_text | TEXT | diff 内容（截断） |
| need_review | INTEGER | 是否需要审查 |

#### review_issues — 审查问题
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | |
| project_name | VARCHAR | |
| revision | VARCHAR | |
| level | VARCHAR | P1/P2/P3/P4 |
| issue_type | VARCHAR | merge_conflict/style/debug... |
| file_path | TEXT | |
| description | TEXT | 问题描述 |
| suggestion | TEXT | 修复建议 |
| engine_type | VARCHAR | rule / llm |
| status | VARCHAR | Open/Resolved |

#### daily_reports — 每日报告
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | |
| report_date | VARCHAR | |
| repo_count / commit_count / file_count / author_count | INTEGER | 统计 |
| p1~p4_count | INTEGER | 各级别问题数 |
| report_markdown | TEXT | 完整报告 |
| teams_push_status | VARCHAR | 推送状态 |

#### author_stats — 提交人统计
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | |
| report_id | INTEGER FK | 关联 daily_reports |
| author | VARCHAR | |
| commit_count | INTEGER | |
| p1~p4_count | INTEGER | |
| issue_density | FLOAT | 问题密度 |

## 6. 配置设计

### 6.1 `config.yaml`

```yaml
timezone: Asia/Shanghai

schedule:
  enabled: true
  hour: 18
  minute: 0

scan:
  max_diff_chars_per_file: 12000
  include_extensions: [".c", ".h", ".cpp", ".hpp", ".py"]
  exclude_paths: ["third_party/", "vendor/", "build/"]
  retry:
    max_retries: 3
    delay: 5
    backoff: 2

projects:
  - name: "项目A"
    svn_url: "file:///F:/SVN%20Store/trunk"
    enabled: true
    owner_group: "研发一组"
    teams_webhook_url: ""
```

## 8. 部署方案

### 8.1 开发模式

```
dev.bat
  ├── 启动 uvicorn (reload, port 8000)
  └── 启动 Next.js dev server (HMR, port 3000, /api/* → proxy to :8000)
```

### 8.2 构建发布

```
构建发布.bat
  ├── cd frontend && npm run build → out/
  ├── 清空 app/static/
  └── 复制 out/* → app/static/
```

### 8.3 生产启动

```
StartTool.bat
  ├── 检查 .env / .venv（自动创建）
  ├── 打开浏览器 http://127.0.0.1:8000
  └── python -m app.main   ← 仅需 Python，无需 Node.js
```

### 8.4 日志与维护

- 日志文件：`logs/app.log`，每天自动轮转，保留 30 天
- 数据库：`data/svn_ai_review.db`，备份此文件即可
- 配置备份：`config.yaml` + `.env`
