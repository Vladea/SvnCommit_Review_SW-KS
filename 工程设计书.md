# SVN AI Review V2.0 — 工程设计书

## 1. 项目概述

### 1.1 背景

SVN AI Review 是一个面向 Windows 内网环境的 SVN 代码自动审查工具，通过定时扫描 SVN 仓库提交记录，提取变更差异，结合规则引擎与 AI 大模型进行自动化代码审查，生成每日审查报告并通过 Teams/Email 等渠道推送。

### 1.2 版本变化

| 维度 | V1.4 | V2.0 |
|------|------|------|
| 架构 | 单文件 297 行 | 模块化分层 26+ 文件 |
| 审查引擎 | 规则 + mock LLM | 规则可开关 + 多 AI 接口可插拔 |
| 前端 | 原生 HTML/JS/CSS | Next.js 14 + Tailwind CSS |
| 日志 | 无 | TimedRotatingFileHandler，30 天保留 |
| 容错 | 无重试 | SVN 操作可配置重试 |
| 通知 | Teams webhook | Teams + Email |
| 设置 | 手动编辑 yaml | 全局 UI 设置页 |
| 定时任务 | 单一 18:00 | 多时段 + 每时段独立通知开关 |
| 部署 | `.bat` 启动 | dev / 构建发布 / StartTool 三脚本模式 |

### 1.3 约束条件

- **部署环境**：Windows 内网单机，仅需 Python + SVN CLI，无 Node.js 运行时
- **数据库**：SQLite，单机部署足够
- **SVN 认证**：必填（内网 SVN 服务器需用户名密码）
- **网络**：内网环境，支持本地 Ollama 模型作为 AI 备选

## 3. 架构设计

### 3.1 分层架构

```
┌──────────────────────────────────────┐
│  前端 (Next.js 静态导出)              │
│  app/static/ ← npm run build 产物   │
├──────────────────────────────────────┤
│  路由层 (app/routes/)                 │
│  dashboard / projects / scan /       │
│  schedule / reports / issues /       │
│  authors / settings                  │
├──────────────────────────────────────┤
│  服务层 (app/services/)               │
│  svn_client / scanner / review /     │
│  report_builder / notifier           │
├──────────────────────────────────────┤
│  数据层 (app/models/ + app/database)  │
│  SQLAlchemy ORM + SQLite            │
├──────────────────────────────────────┤
│  基础设施                              │
│  config / logging / scheduler        │
└──────────────────────────────────────┘
```

### 3.2 数据流

```
[定时触发 / UI手动] → scanner.run_range()
  → svn_client.svn_logs()            # SVN log 获取（含重试）
  → svn_client.filter_by_date()      # 按真实 commit 日期过滤
  → svn_client.svn_diff_safe()       # 获取 diff（含重试）
  → svn_client.split_diff()          # 拆分多文件 diff
  → svn_client.should_review()       # 扩展名/路径过滤
  → rule_engine.rule_review()        # 规则审查（4 条规则可开关）
  → llm_client.llm_review()          # AI 审查（多提供商可插拔）
  → report_builder.create_report()   # Markdown 报告构建
  → notifier.send_notification()     # Teams + Email 推送
```

| 模块 | 文件数 | 职责 |
|------|--------|------|

| 数据库 | `app/database.py` | engine、SessionLocal、Base、init_db |
| 日志 | `app/logging_config.py` | TimedRotatingFileHandler（30天）+ 控制台 |

| 模型 | `app/models/` (6) | SvnCommit、ChangedFile、ReviewIssue、DailyReport、AuthorStat |
| Schema | `app/schemas/` (4) | ProjectRequest、ScanRangeRequest、ScheduleRequest、LLM 相关 |
| SVN | `app/services/svn_client.py` | svn log/diff/split/filter，可配置重试 |

| 路由 | `app/routes/` (8) | dashboard、projects、scan、schedule、reports、issues、authors、settings |

## 4. 数据库设计

### 4.1 E-R 关系

```
svn_commits (1) ──── (N) changed_files
      │
      │ (project_name + revision)
      ↓
review_issues

daily_reports (1) ──── (N) author_stats
```

## 6. 配置设计

### 6.1 `config.yaml`（完整示例）

```yaml
timezone: Asia/Shanghai

schedule:
  entries:
    - hour: 9
      minute: 0
      enabled: true
      notify_teams: true
      notify_email: false
    - hour: 18
      minute: 0
      enabled: true
      notify_teams: true
      notify_email: true

scan:
  max_diff_chars_per_file: 12000
  include_extensions:
    - ".c" / ".h" / ".cpp" / ".hpp" / ".py" / ...
  exclude_paths:
    - "third_party/" / "vendor/" / "build/" / ...
  rules:
    merge_conflict: true
    todo_marker: true
    debug_print: true
    memory_safety: true
  retry:
    max_retries: 3
    delay: 5
    backoff: 2

llm:
  default: "deepseek_free"
  fallback: ""
  concurrent: 3
  retry_count: 2
  retry_delay: 5
  providers:
    - id: deepseek_free
      name: "DeepSeek Chat（免费）"
      type: openai_compatible
      api_base: "https://api.deepseek.com/v1"
      api_key_ref: "DEEPSEEK_API_KEY"
      model: "deepseek-chat"
      enabled: false

notifications:
  teams:
    enabled: true
    webhook_url_ref: "TEAMS_WEBHOOK_URL"
  email:
    enabled: false
    smtp_host: ""
    smtp_port: 587
    smtp_user: ""
    smtp_password_ref: "EMAIL_SMTP_PASSWORD"
    from_addr: ""
    to_addrs: []

projects:
  - name: "项目A"
    svn_url: "file:///F:/SVN%20Store/trunk"
    enabled: true
    owner_group: "研发一组"
```

## 8. 部署方案

### 8.1 模式一览

| 模式 | 启动方式 | 前端端口 | 说明 |
|------|----------|---------|------|
| 开发 | `dev.bat` | `:3000` (Next.js HMR) | 前后端热更新，API 自动代理 |
| 构建 | `构建发布.bat` | — | npm build → app/static/ |
| 生产 | `StartTool.bat` | `:8000` (FastAPI) | 仅 Python，无 Node.js |

### 8.2 文件清单

| 路径 | 用途 | 版本控制 |
|------|------|---------|
| `app/*.py` | 后端源码 | ✅ 提交 |
| `app/static/` | 前端构建产物 | ✅ 提交（Release 包含） |
| `frontend/src/` | 前端源码 | ✅ 提交 |
| `frontend/node_modules/` | 前端依赖 | ❌ gitignore |
| `frontend/.next/` `out/` | 编译中间产物 | ❌ gitignore |
| `config.yaml` | 运行配置 | ✅ 提交 |
| `.env` | 密钥 | ❌ gitignore |
| `.env.example` | 密钥模板 | ✅ 提交 |
| `data/` `reports/` `logs/` | 运行时数据 | ❌ gitignore |
| `.venv/` `__pycache__/` `*.pyc` | Python 运行时 | ❌ gitignore |

### 8.3 日志与维护

- 日志：`logs/app.log`，每天自动轮转，保留 30 天，自动清理
- 数据库：`data/svn_ai_review.db`，备份此文件
- 配置：备份 `config.yaml` + `.env`
